# 资产管理OA系统需求方案书

## 1. 项目概述

### 1.1 项目背景
随着交通集团资产规模的不断扩大，资产管理工作面临着日益复杂的挑战。传统的资产管理方式已经无法满足现代化管理的需求，需要建立一个集中化、规范化、智能化的资产管理OA系统，实现资产的全生命周期管理。

### 1.2 项目目标
- 建立统一的资产管理平台，实现资产信息的集中管理
- 优化资产管理流程，提高管理效率和决策水平
- 实现资产的动态监控和实时统计分析
- 支持多维度的资产报表生成和数据导出
- 建立完善的用户权限管理体系

### 1.3 系统架构
系统采用HTML+Tailwind CSS构建现代化界面，使用模块化JavaScript实现交互功能，通过iframe嵌套实现页面切换。系统采用响应式设计，适配桌面端和移动端设备。

## 2. 功能需求

### 2.1 系统主页 (index.html)

#### 功能模块
- **系统导航**：左侧固定导航菜单，包含系统所有功能模块入口
- **用户信息**：顶部显示当前登录用户信息，包括用户名和邮箱
- **内容区域**：使用iframe加载各功能页面，实现无刷新页面切换
- **响应式设计**：适配不同屏幕尺寸，移动端支持菜单折叠

#### 导航菜单结构
- 首页管理
- 单位管理
- 资产信息（子菜单：各单位列表）
- 报表统计（子菜单：统计报表、资产统计）
- 用户管理
- 角色管理

### 2.2 首页管理 (home.html)

#### 功能模块
- **资产概览**：显示核心统计指标卡片
  - 总资产数量：各单位资产数量总和
  - 出租率：已出租资产数量 ÷ 总资产数量 × 100%
  - 闲置资产：出租属性且状态为"闲置中"的资产数量
  - 盘活资产：上一周期出租属性闲置资产数量 - 本周期出租属性闲置资产数量

- **筛选功能**：支持按年度、季度、月度进行数据筛选
  - 年度筛选：选择具体年份（2022-2026年）
  - 季度筛选：选择具体季度（Q1-Q4）
  - 月度筛选：选择具体月份（1-12月）

- **图表展示**：
  - 资产出租率趋势图：展示不同周期的出租率变化
  - 资产状态分布图：展示当前资产状态分布情况

- **最近资产动态**：展示最近更新的资产信息，包括资产名称、所属单位、状态、更新时间等

#### 交互功能
- 点击统计卡片可查看详细资产信息
- 问号图标悬停显示计算公式和备注说明
- 点击问号图标弹出详细说明模态框
- 筛选条件变更后实时更新统计数据和图表

### 2.3 单位管理 (units.html)

#### 功能模块
- **单位列表管理**：
  - 显示单位卡片列表，包含单位基本信息
  - 支持单位名称模糊搜索
  - 添加新单位功能

- **单位详情**：
  - 单位基本信息展示（名称、类型、负责人、联系电话等）
  - 单位资产统计信息（总资产数量、出租率、闲置资产等）
  - 创建单位管理员账号功能

- **资产列表管理**：
  - 显示单位所属资产列表
  - 支持资产名称搜索和状态筛选
  - 添加新资产功能

#### 添加资产表单字段
- 资产名称（必填）
- 资产类型（必填）：房屋建筑、土地、设备
- 面积
- 出租属性（必填）：出租、非出租
- 资产状态（必填）：已出租、闲置中、维修中、自用
- 支付周期（条件显示）：月付、季付、年付
- 租金金额（条件显示）

#### 交互逻辑
- 选择出租属性时，动态显示支付周期和租金金额字段
- 单位卡片支持点击查看详情
- 资产列表支持分页显示
- 搜索功能实时过滤单位和资产

### 2.4 资产统计 (asset_stats.html)

#### 功能模块
- **多维度筛选**：
  - 单位筛选：全部单位或指定单位
  - 资产类型筛选：房屋建筑、土地、设备设施、其他
  - 资产状态筛选：已出租、闲置中、维修中、自用
  - 出租属性筛选：出租属性、非出租属性

- **资产列表展示**：
  - 按单位分组展示资产信息
  - 支持单位展开/收起功能
  - 显示资产详细信息（名称、类型、面积、状态、出租属性、租金）

- **批量操作**：
  - 全部展开/收起功能
  - 导出资产列表功能

#### 交互功能
- 筛选条件变更时实时更新资产列表
- 点击单位标题可展开/收起该单位的资产列表
- 支持批量操作提高管理效率

### 2.5 报表统计 (reports.html)

#### 功能模块
- **报表类型切换**：
  - 年度报表：全年数据统计（1月1日-12月31日）
  - 半年度报表：上半年数据统计（1月1日-6月30日）
  - 季度报表：具体季度数据统计（如Q1:1-3月）

- **时间筛选**：
  - 年份选择：2022-2024年
  - 季度选择：Q1-Q4（仅季度报表显示）

- **核心指标卡片**：
  - 总资产数量及同比增长
  - 出租率及同比提升
  - 盘活资产数量及同比变化

- **详细数据报表**：
  - 年度报表：显示2024年vs2023年的各项指标对比
  - 半年度报表：显示2024年上半年vs2023年上半年的对比
  - 季度报表：显示当前季度vs上一季度的环比对比

#### 计算公式
- 出租率 = 已出租资产数量 ÷ 出租属性资产总数量 × 100%
- 闲置率 = 闲置资产数量 ÷ 总资产数量 × 100%
- 盘活资产数量 = 去年出租属性的闲置资产 - 今年出租属性的闲置资产
- 盘活资产转化率 = 出租属性的盘活资产 ÷ 出租属性的闲置资产总量

### 2.6 用户管理 (users.html)

#### 功能模块
- **用户列表管理**：
  - 显示用户基本信息（单位、账号、姓名、联系电话、权限等级、状态）
  - 支持按账号、单位、状态进行筛选
  - 支持账号搜索功能

- **用户操作**：
  - 创建账号功能
  - 编辑用户信息功能
  - 启用/禁用账号功能

- **权限控制**：
  - 超级管理员：可管理所有用户和角色
  - 单位管理员：只能管理本单位用户，且只能创建操作员和查看员角色

#### 用户角色体系
- 超级管理员：拥有系统所有权限
- 单位管理员：管理本单位的资产和用户
- 单位操作员：操作本单位资产
- 单位查看员：查看本单位资产

### 2.7 角色管理 (roles.html)

#### 功能模块
- **角色列表管理**：
  - 显示角色基本信息（名称、描述）
  - 支持角色搜索功能

- **角色操作**：
  - 创建角色功能（仅超级管理员）
  - 编辑角色功能（仅超级管理员）
  - 删除角色功能（仅超级管理员，系统内置角色不可删除）

- **权限设置**：
  - 首页管理权限：查看首页统计、导出统计数据
  - 单位管理权限：查看单位列表、添加单位、编辑单位、删除单位
  - 资产信息权限：查看资产列表、添加资产、编辑资产、删除资产、导入资产、导出资产
  - 报表统计权限：查看报表、导出报表、打印报表
  - 用户管理权限：查看用户列表、添加用户、编辑用户、删除用户、重置密码
  - 角色管理权限：查看角色列表、添加角色、编辑角色、删除角色

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间不超过2秒
- 支持至少100个并发用户访问
- 数据查询响应时间不超过1秒

### 3.2 安全性需求
- 用户认证和授权机制
- 数据传输加密
- 防止SQL注入和XSS攻击
- 定期数据备份

### 3.3 可用性需求
- 系统可用性达到99.9%
- 支持7×24小时运行
- 提供友好的错误提示和异常处理

### 3.4 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 响应式设计，适配桌面端和移动端
- 支持不同屏幕分辨率

### 3.5 可维护性需求
- 模块化设计，便于扩展和维护
- 清晰的代码结构和注释
- 完善的文档支持

## 4. 技术实现

### 4.1 前端技术栈
- HTML5 + CSS3 + JavaScript
- Tailwind CSS：现代化UI框架
- FontAwesome：图标库
- Chart.js：数据可视化图表
- 响应式设计：适配不同设备

### 4.2 系统架构
- 单页面应用架构
- iframe嵌套实现页面切换
- 模块化JavaScript设计
- 动态DOM操作实现内容更新

### 4.3 数据管理
- 前端数据模拟（实际项目中应连接后端数据库）
- 本地存储用于临时数据缓存
- 表单验证和数据校验

## 8. 结论

本需求方案书详细描述了资产管理OA系统的功能需求、非功能需求、技术实现方案和项目实施计划。系统采用现代化的前端技术栈，实现了资产的全生命周期管理，支持多维度的数据统计和分析。通过建立完善的用户权限管理体系，确保系统的安全性和可靠性。

该系统的实施将显著提高资产管理效率，为决策提供数据支持，实现资产管理的规范化和智能化。
